# AGX 1T 硬盘挂载工具

> **注意：迁移前请做好重要数据备份！**

将 `/home` 目录迁移到 1T 固态硬盘的自动化脚本。

## 功能说明

该脚本完成以下操作：

1. 备份现有 fstab 配置
2. 检测/创建 GPT 分区并格式化为 ext4
3. 将现有 `/home` 数据迁移到 1T 硬盘
4. 配置开机自动挂载

## 使用方法

### 1. 运行脚本

```bash
sudo ./tools/mount_tools/create_and_format_partition.sh
```

### 2. 等待迁移完成

脚本会自动执行以下步骤：
- 备份 fstab
- 创建/检测分区
- 格式化分区
- 迁移 /home 数据
- 配置自动挂载

### 3. 重启生效

```bash
sudo reboot
```

## 注意事项

### 运行前
- **必须使用 sudo 运行**
- 确保已备份重要数据
- 建议关闭所有使用 /home 的程序（IDE、终端、文件管理器等）
- 停止所有 Docker 容器，避免数据不一致
- 建议使用 SSH 连接运行脚本，避免图形界面占用

### 运行中
- 首次运行会复制所有 /home 数据，可能需要几分钟到十几分钟
- 脚本会自动验证数据完整性
- 如果有文件权限问题，脚本会提示并等待确认
- 迁移完成后会自动清理备份文件

### 迁移后清理
如果迁移后发现备份目录未删除干净，在重启后可执行：

```bash
sudo fuser -k /home.backup_* 2>/dev/null
sudo rm -rf /home.backup_*
```

## 配置参数

如需修改默认配置，可编辑脚本开头的变量：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| DISK | /dev/nvme0n1 | 硬盘设备名 |
| PARTITION | ${DISK}p1 | 分区名 |
| HOME_MOUNT | /home | 挂载点 |


## 验证挂载

重启后执行以下命令验证：

```bash
df -h | grep /home
lsblk | grep nvme
```
